12. Java 제네릭

Java 제네릭

제네릭 클래스 접근지정자 class 클래스명 <>{ }

제네릭 인터페이스 접근지정자 interface 클래스명<>{ }

제네릭 은 객체를 생성할 때 제네릭 타입 변수에 실제 타입을 대입하는 것

제네릭 은 클래스를 정의하는 시점에 타입을 지정하는 것이 아니라 객체를 생성하는 시점에 타입을 지정

제네릭 메서드 는 호출되는 시점에 실제 제네릭 타입을 지정

제네릭 메서드의 문법 구조 접근지정자 <T,V> T 메서드명 (T t, int a) { }

제네릭 메서드의 호출 문법 구조 참조객체.<실제제네릭타입>메서드명(입력매개변수);

제네릭 메서드 내에서 사용할 수 있는 메서드 는 Object만 가능하다. (후에 제네릭 타입 변수가 확정 되더라도 사용할 수 있어야 해서)

제네릭 클래스의 타입 제한 접근지정자 class 클래스명<T extends 클래스/인터페이스>{ }
*클래스/인터페이스 상관없이 항상 extends 사용

제네릭 메서드의 타입 제한 접근지정자 <T extends 클래스/인터페이스> T 메서드명(T t) { }

메서드 매개변수일 때 제네릭

리턴타입 메서드명(제네릭 클래스명<?> 참조변수명){ }
리턴타입 메서드명(제네릭 클래스명<? extends 클래스/인터페이스> 참조변수명){ }

void method(Goods<A> v) A인 객체만 가능
void method(Goods<?> v) 모든 타입 객체 가능
void method(Goods<? extends B > v) B와 B를 상속받은 클래스 객체만 가능
void method(Goods<? super B> v) B와 B의 부모 클래스인 객체만 가능

제네릭 클래스의 상속 자식 클래스의 제네릭 타입 변수의 개수는 항상 부모보다 같거나 많다.

제네릭 메서드의 상속 부모 클래스 내의 제네릭 메서드는 자식 클래스 또한 제네릭을 그대로 물려받음
