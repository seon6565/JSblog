11. Java exception

Java exception

exception 개발자가 해결할 수 있는 오류(차선책 선택)

error 개발자가 해결할 수 없는 오류

일반 예외(Checked Exception) 컴파일 전에 체크

실행 예외(Unchecked Exception=Runtime Exception) 실행할 때 체크

Exception, RuntimeException

예외 처리

예외 처리 문법 구조 try { 실행 } catch(예외클래스명 참조변수명){ 예외처리 } finally{ 예외여부 상관없이 무조건 실행(생략가능)}

다중 예외 처리 try{} catch(예외1 참조변수명){} catch(예외2 | 예외3 참조변수명){} 작은 범위에서 큰 범위로 작성할 것

리소스 해제 예외 처리 finally 블록의 대표적인 기능은 리소스를 해제하는 것
try{} catch(Exception e){} finally{ 자원반납 }

리소스 자동 해제 예외 처리
try(리소스 자동 해제가 필요한 객체 생성){} catch(예외클래스명 참조변수명){}
try(InputStreamReader is = new InputStreamReader(System.in);){ is.read(); } catch(Exception e){ 예외처리 }
해당 InputStreamReader는 AutoCloseable 인터페이스가 구현되어 있어서 자동으로 close()가 호출된다.

AutoCloseable 인터페이스 리소스 자동 해제 객체는 반드시 close메서드를 포함하고 AutoCloseable 인터페이스를 구현 해야 함

예외 전가 리턴타입 메서드명() throws 예외클래스명{ }
예외를 전가하면 예외 처리의 의무를 호출한 곳으로 전가 된다.(최종적으로는 JVM이 처리함)

다중 예외 전가 리턴타입 메서드명() throws 예외클래명A,....,예외클래스명D{ }

printStackTrace() 예외 발생이 전달되는 경로를 한눈에 확인할 수 있는 메서드
