13. Java 컬렉션

Java 컬렉션

컬렉션 은 동일한 타입을 묶어 관리하는 자료구조

컬렉션 프레임워크 컬렉션의 설계된 프레임워크들 이다.

List<E> 는 인터페이스이다.

List<E>의 주요 메서드
add(E), add(int, E), addAll(Collection), addAll(int, Collection)
set(int, E)
remove(int), remove(Object), clear()
get(int), size(), isEmpty()
toArray(), toArray(T[])

ArrayList<E> 는 List<E>의 대표적인 구현 클래스로 Element를 index로 관리하며 저장 용량(기본값10)을 동적 관리한다.

Vector<E> 는 ArrayList<E>와 유사하지만 주요 메서드가 동기화 메서드로 구현되어 있다.
*멀티 쓰레드에서 사용할 수 있도록 기능이 추가된 것

LinkedList<E> 는 저장용량을 지정할 수없으며, index가 아닌 앞뒤 객체의 정보를 저장한다.
추가, 삭제는 LinkedList<E>가 빠르지만 검색의 경우 ArrayList<E>가 빠르다.

Set<E> 는 인터페이스이다.

Set<E>의 주요 메서드
add(E), addAll(Collection)
remove(Object), clear()
isEmpty(), contains(Object), size(), iterator()
toArray(), toArray(T[])

HashSet<E> 는 대표적인 Set<E>의 구현 클래스로 순서 없이 저장 용량(기본값16)을 동적 관리하며, 중복을 허용하지 않는다.

iterator() 메서드를 호출하면 Iterator<E>의 객체가 생성 된다.
hasNext() 다음으로 가리킬 원소의 존재 여부를 boolean으로 리턴
next() 다음 원소 위치로 가서 읽은 값을 리턴

해시코드 객체가 저장된 번지를 기준으로 생성된 정수형 고윳값

HashSet<E> 에서의 중복 확인 메커니즘
1. hashcode() 동일한지 확인
2. equal() 결과가 true인지 확인

LinkedHashSet<E> 출력 순서가 항상 입력 순서와 동일(데이터 간의 연결 정보를 추가로 갖고 있음)

TreeSet<E> 크기 순으로 출력 (정렬과 검색 기능이 추가된 것)

Map<K,V> List<E>와 Set<E>는 Collection<E> 인터페이스를 상속받지만 Map<K,V>는 별도의 인터페이스로 존재함.

Map<K,V> 는 Key와 Value 한 쌍으로 데이터를 저장(1쌍=entry)

Key 는 중복 저장 불가

Value 는 중복 저장이 가능

Map<K,V>의 주요 메서드
put(key, value), putAll(Map)
replace(key, value), replace(key, oldvalue, newvalue)
get(key), containsKey(key), containsValue(value), keySet()=Set객체로 리턴, entrySet()=Set객체로 리턴, size()
remove(key), remove(key, value), clear()

HashMap<K,V> Map<K,V>의 대표적인 구현 클래스로 Key값의 중복 여부를 확인하는 메커니즘은 HashSet<E>과 동일하다.
초기 저장 용량은 16이며, 입출력 순서는 동일하지 않을 수도 있다.

Hashtable<K,V> HashMap에 멀티 쓰레드에 안전한 특징을 추가한 것이다.

LinkedHashMap<K,V> HashMap에 입력 데이터의 순서 정보를 추가로 갖고 있는 것이다.

TreeMap<K,V> 입력 순서와 상관없이 데이터를 Key값의 크기 순으로 저장한다.

Stack<E> last in first out의 Stack 자료구조를 구현한 컬렉션이다.

Stack<E>의 주요 메서드
push(item), peek(), search(Object), pop(), empty()
*Vector<E>의 자식 클래스이므로 Stack<E>의 메서드도 사용이 가능하나 LIFO의 특성이 반영되지 않는다.

Queue<E> first in first out의 Queue 자료구조를 구현한 컬렉션이다.

Stack<E>의 주요 메서드
예외 처리 기능 미포함 메서드 add(item), element()=가장 상위 엘리먼트 리턴(데이터 변화X), remove()=가장 상위 엘리먼트 꺼내기
예외 처리 기능 포함 메서드 offer(item), peek(), poll()
