9. Java abstract class, interface

Java abstract class, interface

abstract class 는 내부의 미완성 메서드 떄문에 객체를 직접 생성할 수 없다.

abstract class 를 상속하는 자식 클래스는 반드시 abstract method를 완성(오버라이딩)해야 한다.

abstract class 의 객체 생성 방법

1. 일반 클래스로 상속해서 객체 생성 상속 받은 클래스에서 메서드 오버라이딩

2. 익명 이너 클래스 클래스명 참조변수명 = new 생성자() { 추상 메서드 오버라이딩 };

interface 는 내부의 모든 필드가 public static final, 메서드는 public abstract로 정의됨
*static, default 메서드는 제외

interface의 구조 interface 인터페이스명 { public static final 자료형 필드명 = 값; public abstract 리턴타입 메서드명(); }

interface의 상속 상속 클래스명 implements 인터페이스명, ..... 인터페이스명 { 내용 }

interface의 클래스와 인터페이스 동시에 상속 클래스명 extends 클래스명 implements 인터페이스명, ....... 인터페이스명 { 내용 }
*interface는 클래스 상속 불가능, interface가 interface를 상속받을 경우 extends

interface 의 객체 생성 방법

1. 일반 클래스로 상속해서 객체 생성 상속 받은 클래스에서 메서드 오버라이딩

2. 익명 이너 클래스 사용 클래스명 참조변수명 = new 생성자() { 추상 메서드 오버라이딩 };

interface의 default 메서드 public default 리턴타입 메서드명() { 완성된 메서드 }
*완성된 메서드이긴 하지만 객체를 생성할수 없기 떄문에 일단 상속 받아야 한다.

interface의 default 메서드 호출 방법 인터페이스명.super.dafault메서드명();

interface의 static 메서드 인터페이스명.static메서드명();
*객체를 생성하지 않고 바로 사용 가능
