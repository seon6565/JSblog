11. 트랜잭션, 커서, 저장 프로시저

트랜잭션, 커서, 저장 프로시저

트랜잭션 한꺼번에 수행되야 할 쿼리문의 묶음(동시성 보장, 무결성)

Atomicity(원자성) 쪼갤수 없는 최소 단위로 이뤄져야함.

Consistency(일관성) 제약이 보장 되야함.

Isolation(독립성) 다른 트랜잭션에 영향을 받으면 안됨

Durability(지속성) 결과가 저장되어 지속적이여야함.

START TRANSACTION; COMMIT; ROLLBACK;

커서 데이터베이스 조회할때 위치(커서)를 의미

DECLARE 커서명 CURSOR FOR SELECT문;

DECLARE CONTINUE HANDLER FOR NOT FOUND SET CHECK = 'Y';

OPEN CURS;

FETCH CURS INTO 변수;

CLOSE CURS;

저장 프로시저

DELIMITER $$ CREATE[ OR REPLACE] PROCEDURE 프로시저명([IN,OUT]) BEGIN END $$ DELIMITER ; *저장 프로시저에서는 NOT ATOMIC 안해도 됨

CALL 프로시저명(매개변수)

DROP PROCEDURE 프로시저이름

SHOW PROCEDURE STATUS 프로시저 어느DB,이름,TYPE 등 조회(내용은 미포함)

SELECT * FROM INFORMATION_SCHEMA.ROUTINES TYPE PROCEDURE에서 내용 포함해서 조회 가능
