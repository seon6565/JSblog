7. JOIN

JOIN

SELF JOIN -> INNER JOIN -> OUTER JOIN -> CROSS JOIN 순서로 데이터베이스에 부하가 걸림

SELECT 칼럼명 FROM 테이블명 JOIN 테이블명2 ON 조인조건 JOIN만 명시할 시 INNER JOIN이 됨

CROSS JOIN 테이블1행 X 테이블2행 으로 합쳐지며 조심해서 사용할 것

INNER JOIN 조인 조건에 맞는 교집합

LEFT OUTER JOIN 왼쪽테이블 기준으로 오른쪽 테이블에 해당 값이 없어도 조건에 맞는 왼쪽테이블의 값은 전부 출력(부분집합)
*해당하는 값이 없는 것들은 NULL처리

RIGHT OUTER JOIN 오른쪽테이블 기준으로 왼쪽 테이블에 해당 값이 없어도 조건에 맞는 오른쪽테이블의 값은 전부 출력(부분집합)
*해당하는 값이 없는 것들은 NULL처리

OUTTER JOIN의 FROM기준 비즈니스 로직의 메인이 FROM에 오게 구성

SELF JOIN 자기 자신과 조인 조건으로 TABLE1과 TABLE1을 비교할떄 사용

NESTED JOIN 다중 조인

서브쿼리 일반적으로 서브쿼리의 결과값은 1개행 1개열로 구성
*서브쿼리에서는 SELECT FROM WHERE만 사용할 것

WHERE NOT EXISTS(서브쿼리) 서브쿼리 결과값이 FALSE면 조회

WHERE EXISTS(서브쿼리) 서브쿼리 결과값이 TRUE면 조회

SHOW PROFILES 날린 쿼리의 시간, 쿼리를 보여 줌

SHOW PROFILES FOR QUERY 숫자 쿼리 숫자에 해당하는 쿼리의 STATUS와 DURATION을 보여줌

EXPLAIN SELECT SELECT 구문의 실행계획을 무거운 순서대로 보여 줌

DELETE 서브쿼리 DELETE FROM TABLE WHERE 칼럼 IN (서브쿼리)

INSERT INTO TABLE(칼럼...) SELECT 칼럼... FROM 테이블

SELECT 구문1 UNION SELECT 구문2 구문1의 결과와 구문2의 결과를 합침(칼럼특징은 같아야하며 UNION ALL을 할 경우 중복도 결과)

SELECT 구문1 EXCEPT SELECT 구문2 구문1의 결과와 구문2의 결과를 빼줌(칼럼특징은 같아야함)

SELECT 구문1 INTERSECT SELECT 구문2 구문1의 결과와 구문2의 결과의 중복값을 보여줌(칼럼특징은 같아야하며 INTERSECT ALL을 할 경우 중복도 결과)
